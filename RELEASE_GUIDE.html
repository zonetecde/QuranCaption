📦 Quran Caption Release Guide

Step-by-step instructions for releasing a new version with auto-update

build One-Time Setup DO THIS ONCE

Before your first release with auto-update, you need to set up the signing key as a GitHub Secret. Since this feature was added via a Pull Request, you (the repository owner) should generate your own unique signing key.

warning
Important: The private signing key must be added as a GitHub Secret. Without it, the CI build will fail because it cannot sign the update artifacts.

Steps for the Repository Owner:

  1. Open your terminal in the project root and run:
    npm run tauri signer generate -- -w ~/.tauri/qurancaption.key
    Press Enter when asked for a password (leave it empty).
  2. This generates two files in ~/.tauri/ (or your user's .tauri folder):
    • qurancaption.key (Private key - KEEP SECRET)
    • qurancaption.key.pub (Public key - safe to share)
  3. Open src-tauri/tauri.conf.json. Under plugins.updater.pubkey, replace the existing public key with the contents of your new qurancaption.key.pub file. Commit and push this change.
  4. Go to GitHub → Repository Settings → Secrets → Actions
  5. Click "New repository secret"
  6. Set the name to TAURI_SIGNING_PRIVATE_KEY and paste the full content of the qurancaption.key (the private key file) as the value.
  7. Click "Add secret". That's it! You only need to do this once.
info
The public key is embedded in tauri.conf.json and ships with every app build. The private key lives only in GitHub Secrets to sign updates.

🚀 How to Release an Update

1

Bump the Version Number

Update the version in three files. All three must match exactly.

📂 QuranCaption/
├── package.json → "version": "3.4.0"
├── 📂 src-tauri/
│ ├── tauri.conf.json → "version": "3.4.0"
│ └── Cargo.toml → version = "3.4.0"
warning
The version must follow SemVer format: MAJOR.MINOR.PATCH (e.g., 3.4.0). No "v" prefix in the config files.
2

Commit & Push to main

Commit your version bump and all code changes, then push to the main branch.

git add -A
git commit -m "release: v3.4.0"
git push origin main
3

Trigger the Build Workflow

Go to GitHub Actions and manually trigger the build:

  1. Navigate to Actions → "Build & Release"
  2. Click "Run workflow" on the right side
  3. Enter the tag: v3.4.0 (with the "v" prefix!)
  4. Choose whether to create it as a draft (recommended: yes)
  5. Click "Run workflow"
info
The workflow builds for Windows, macOS Intel, macOS ARM, and Linux simultaneously. It typically takes 20–45 minutes.
4

Wait for CI to Complete

Monitor the workflow run. Once all 4 platform builds succeed, a GitHub Release draft will be created automatically with all the assets, including:

📦 Release Assets (auto-generated):
├── Quran Caption_3.4.0_x64-setup.exe    ← Windows installer
├── Quran Caption_3.4.0_x64-setup.exe.sig
├── Quran Caption_3.4.0_aarch64.dmg     ← macOS ARM
├── Quran Caption_3.4.0_x64.dmg         ← macOS Intel
├── Quran Caption_3.4.0_amd64.AppImage  ← Linux
├── Quran Caption_3.4.0_amd64.AppImage.sig
├── Quran Caption.app.tar.gz
├── Quran Caption.app.tar.gz.sig
└── latest.json                       ← Auto-update manifest!
5

Edit & Publish the Release

If you created the release as a draft:

  1. Go to Releases
  2. Find the draft release (it will have a "Draft" badge)
  3. Click "Edit" → write a proper changelog
  4. Make sure "Set as the latest release" is checked
  5. Click "Publish release"
check_circle
That's it! Once published as "latest", all existing users will see the update prompt in-app and can update with one click.
6

Users Get the Update Automatically ✨

When users open Quran Caption, the app checks for updates by fetching latest.json from the latest GitHub Release. If a newer version is found:

  1. A beautiful modal appears showing the changelog
  2. User clicks "Update Now"
  3. The update downloads with a progress bar
  4. It installs automatically
  5. The app restarts on the new version 🎉

🔧 Troubleshooting

error
Build fails with "TAURI_SIGNING_PRIVATE_KEY not set"
You haven't added the signing private key as a GitHub Secret. Follow the "One-Time Setup" section above.
warning
Users don't see the update
Make sure the release is marked as "latest" (not draft, not pre-release). The app fetches latest.json from the /releases/latest/download/ endpoint which only works for the latest non-draft, non-prerelease release.
info
Version tag format
The tag you enter in the GitHub Action must start with v (e.g., v3.4.0), while the version in the config files should NOT have the v prefix (e.g., 3.4.0).
